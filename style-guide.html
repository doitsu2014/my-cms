07 · Forms

Forms — the form-control + label + label-text + input pattern

Every form in the app wraps inputs in label.form-control with div.label > span.label-text. Errors surface as div.label > span.label-text-alt.text-error. Focus states use focus:input-primary / focus:select-primary.

14 · Patterns

Recurring recipes — the ones that show up in every form page

1. Tinted icon container + title + subtitle

Used in category-form.tsx, blog-form.tsx, modal headers.

<div className="flex items-start gap-4">
  <div className="bg-primary/10 p-3 rounded-xl">
    FolderOpen w-6 h-6 text-primary
  </div>
  <div className="flex-1">
    <h2 className="card-title text-lg">Basic Information</h2>
    <p className="text-sm text-base-content/60">Set the display name…</p>
  </div>
</div>

2. Colored-stripe card (4px top border in theme color)

Each form card uses a different top-stripe color: border-t-primary / border-t-accent / border-t-secondary.

<div className="card bg-base-100 shadow-lg border-t-4 border-t-primary hover:shadow-xl transition-shadow duration-300">

3. Floating Action Button with expandable secondary actions

The Save/Cancel buttons on the form pages are anchored to the bottom-right as a FAB that opens to reveal two action buttons.

// from blog-form.tsx + category-form.tsx
<div className="fixed bottom-8 right-8 z-50 flex flex-col items-center gap-3">
  <!-- expand on click -->
  <button className="btn btn-lg btn-circle btn-primary shadow-lg rotate-45">Plus</button>
</div>

Used in the media grid, categories table, and translation tabs when there's no data yet.
